# Philosophy

Academic data should be hierarchical.

Every lower entity inherits context from its parent.

Never duplicate information.

Everything should be connected.

# Golden Rules

1.

Masters should never contain operational data.

2.

Programs define business offerings.

3.

Subjects define academic content.

4.

Batches define delivery.

5.

Students belong to Batches.

6.

Attendance belongs to Batch + Subject.

7.

Faculty teaches Subjects inside Batches.

8.

Everything must be reusable.

Never duplicate.

Always map.
